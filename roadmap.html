Chiến Dịch: 30 ngày Chinh Phục DevOps

Giai đoạn 1: Chiếm lĩnh "Cục Sắt"

Chinh phục Linux & Mạng cơ bản.

Hack tư duy DB:
- Phân quyền Linux (chmod/chown) y hệt như GRANT/REVOKE trong SQL.
- Bỏ chuột, dùng Terminal như cách chị hay gõ Query thay vì dùng GUI.

Giai đoạn 1: Chiếm lĩnh "Cục Sắt"

Chinh phục Linux & Mạng cơ bản.

  • Linux Survival: Bash shell, thao tác file, quản lý Process. Cấm dùng chuột.
  • Networking & SSH: IP, Port. Phân biệt Public/Private Key (như ổ khóa và chìa).
  • Multipass VM: Tạo và xóa máy ảo liên tục để làm quen với "Hạ tầng dùng một lần".

Giai đoạn 2: Đóng gói & Điều binh

Biến hệ thống thành Container bất tử.

  • Docker Core: Đóng gói Postgres/MySQL vào container. Master khái niệm Volume.
  • Docker Compose: Điều phối multi-container (App gọi DB nội bộ).
  • K3s Basics: Làm quen Pod (Lính), Deployment (Tướng), Service (Định tuyến).

Giai đoạn 2: Đóng gói & Điều binh

Biến hệ thống thành Container bất tử.

Hack tư duy DB:
- Chị hiểu rõ dữ liệu cần Persistence (sống sót khi sập máy). Hãy áp dụng tư duy đó vào Docker Volumes. Container chết, Volume giữ data DB sống mãi.

Giai đoạn 3: Lệnh bài Terraform

Infrastructure as Code (IaC).

Hack tư duy DB:
- Terraform State (.tfstate) chính là Transaction Log của cơ sở hạ tầng. Nó ghi nhận sự thật tuyệt đối. Lệnh apply giống như COMMIT một transaction.

Giai đoạn 3: Lệnh bài Terraform

Infrastructure as Code (IaC).

  • Providers & Resources: Dùng code HCL để điều khiển AWS/Multipass tạo máy ảo.
  • The State File: Quản lý tfstate, hiểu cách Terraform tính toán sự thay đổi (plan).
  • Provisioners & Cloud-init: Tự động tiêm script cấu hình ngay khi VM vừa boot xong.

Giai đoạn 4: Tổng tư lệnh GitOps

Tự động hóa toàn diện & Bảo mật cấp cao.

  • ArgoCD (GitOps): Biến GitHub thành Single Source of Truth. ArgoCD tự đồng bộ.
  • Secret Management (Vault): Bảo mật DB credentials. Không hardcode mật khẩu.
  • Final Boss: 1 lệnh terraform apply đẻ ra VM -> Cài K3s/Vault/ArgoCD -> Kéo App & DB lên.

Giai đoạn 4: Tổng tư lệnh GitOps

Tự động hóa toàn diện & Bảo mật cấp cao.

Hack tư duy DB:
- ArgoCD hoạt động như cơ chế Master-Slave Replication. GitHub là Master, Cluster K8s là Slave. Mọi rẽ nhánh sai lệch trên Cluster sẽ bị ArgoCD "ghi đè" để khớp với Master.
